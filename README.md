# Simple Booking System with LINE Alerts

This project was created during my internship to make office reservations easier. It helps staff book what they need and sends instant alerts to the team via LINE, so everyone stays updated.

---

### 1. The Heart of the Project (Purpose)
I built this system to replace slow, manual booking with a fast and friendly digital tool.

| Current Goal | How I helped | Warm & Clear Result |
| :--- | :--- | :--- |
| **Easy Booking** | Simple Web Form | Staff can book everything in just a few clicks |
| **Instant Alerts** | LINE Notify API | Team members get a message on their phones immediately |
| **Safe Records** | Organized Database | No more lost or confusing booking information |
| **Quick Approval** | Admin Dashboard | Leaders can approve or reject requests easily |

---

### 2. How it works for everyone (Simple Design)
The system is designed to be very simple so that anyone can use it without needing a manual.

| Step | What happens | Why it feels good |
| :--- | :--- | :--- |
| **1. Request** | User picks a time and submits | It is fast and very easy to do |
| **2. Check** | The system checks if the slot is free | No more double-bookings or mistakes |
| **3. Notify** | A message is sent to the admin's LINE | The team knows about it right away |
| **4. Reply** | Admin gives an answer | The user gets an update instantly |

---

### 3. Built with Care (Design Focus)
I focused on making this system reliable so that the team can trust it every day.

* **Friendly Design:** Clean and simple web pages for a better experience.
* **Instant Support:** Using LINE because it is the app everyone already loves and uses.
* **Reliability:** Built with smart checks to make sure every booking is accurate.
* **Teamwork:** Designed to help staff and admins work together more smoothly.
